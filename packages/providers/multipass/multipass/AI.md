# @cdk-x/multipass

## Role

Multipass provider for cdkx. Provides L1 constructs for declaring local development VMs via [Canonical Multipass](https://multipass.run/). Resources synthesize YAML configuration files that are committed to the repository — they do **not** deploy anything to a cloud API.

## Architecture

```
@cdk-x/multipass
 └── src/lib/
      ├── generated/
      │    └── resources.generated.ts   ← auto-generated by spec-to-cdkx (DO NOT EDIT)
      └── provider/
           └── provider.ts              ← MltProvider (identifier = 'multipass')
```

Resources auto-generated from JSON Schemas in `schemas/v1/`:
- `MltVm` (`Multipass::VM::Instance`) — a Multipass virtual machine

## Synthesis

`MltVm` is used with `YamlFileSynthesizer` from `@cdk-x/core`. The synthesizer writes one YAML file per root resource to the configured `outputDir`. Child resources (e.g., network attachments) are composed into the parent's YAML output.

## Relationship with other packages

- Depends on `@cdk-x/core` for `ProviderResource`, `IResolvable`, `YamlFileSynthesizer`
- Uses `@cdk-x/spec-to-cdkx` (dev) for codegen
- Consumed by `.cdkxrc.ts` at the repo root, executed by `cdkx project` from `@cdk-x/cli`

## Regenerating L1 constructs

```sh
npx nx run @cdk-x/multipass:codegen
```
